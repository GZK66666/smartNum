# SmartNum v2.0 版本升级文档

## 版本信息

- **版本号**: v2.0
- **发布日期**: 2026-03-08
- **升级类型**: 架构重构

## 升级概述

v2.0 是一次重大架构升级，将系统从**基于手动路由 + 多个专用 Agent**重构为**基于 DeepAgents 的统一自主智能体**架构。

## 架构变化

### v1.1 架构（旧）

```
用户输入 → RouterAgent（意图识别）
              ↓
    ┌─────────┼─────────┐
    ↓         ↓         ↓
SqlAgent  AnalysisAgent  ChitchatAgent
    ↓         ↓         ↓
  结果      分析结果    回复
```

**问题**：
- 架构复杂，需要维护多个 Agent
- 意图识别可能不准确
- 限制了 Agent 的自主性
- 没有充分利用 DeepAgents 的 loop 能力

### v2.0 架构（新）

```
用户输入 → 统一 DeepAgent
              ↓
         [自主循环]
    ┌─────────┼─────────┐
    ↓         ↓         ↓
get_schema  run_sql  generate_chart
    ↓         ↓         ↓
         最终回答
```

**优势**：
- 架构简洁，只有一个智能体
- Agent 自主决定使用哪些工具
- 充分利用 DeepAgents 的 loop 能力
- 可以多步推理和多次工具调用


### 2. 新增的依赖

```txt

```

### 3. 重写的文件

`app/services/agent_service.py` - 完全重写

**核心变化**：
- 创建单一 DeepAgent 单例
- 定义 3 个核心工具：`get_schema`, `run_sql`, `generate_chart`
- 编写统一的 System Prompt
- 实现 SSE 流式输出适配

## 工具定义

### get_schema

获取数据库 Schema 信息。

```python
def get_schema(
    datasource_id: str,
    table_pattern: Optional[str] = None,
) -> str:
    """
    Args:
        datasource_id: 数据源ID
        table_pattern: 表名匹配模式（支持通配符 %）
    Returns:
        数据库表结构信息（Markdown格式）
    """
```

### run_sql

执行 SQL 查询并返回结果。

```python
def run_sql(
    datasource_id: str,
    sql: str,
    limit: int = 1000,
) -> dict:
    """
    Args:
        datasource_id: 数据源ID
        sql: SQL 查询语句（仅支持 SELECT）
        limit: 最大返回行数
    Returns:
        {success, columns, rows, total, truncated, error}
    """
```

### generate_chart

根据数据生成图表配置。

```python
def generate_chart(
    data: dict,
    chart_type: str = "auto",
    title: str = "",
) -> dict:
    """
    Args:
        data: 查询结果数据
        chart_type: 图表类型 (line/bar/pie/scatter/auto)
        title: 图表标题
    Returns:
        {success, chart_type, option}
    """
```

## SSE 事件流

保持与前端兼容的 SSE 事件类型：

| 事件类型 | 说明 |
|---------|------|
| `thinking` | 思考过程 |
| `plan` | 任务规划 |
| `tool_call` | 工具调用开始 |
| `tool_result` | 工具调用结果 |
| `sql_generation` | SQL 生成 |
| `sql_execution` | SQL 执行状态 |
| `visualization` | 可视化建议 |
| `result` | 查询结果 |
| `message` | 文本消息 |
| `error` | 错误信息 |
| `done` | 处理完成 |

## DeepAgents 特性利用

v2.0 充分利用 DeepAgents 的核心能力：

### 1. 自动规划（write_todos）

Agent 在处理复杂任务时，会自动使用 `write_todos` 工具进行任务分解和规划。

### 2. 自主循环

Agent 在 loop 中可以：
- 分析问题
- 调用工具
- 查看结果
- 调整策略
- 继续迭代

直到任务完成。

### 3. 上下文管理

DeepAgents 内置的文件系统工具可以帮助 Agent：
- 管理大型上下文
- 存储中间结果
- 防止上下文溢出
