## 1. 产品概述

### 1.1 产品定位

smartNum 是一款智能问数系统，旨在让非技术用户也能通过自然语言查询数据库，降低数据分析门槛，提升数据驱动决策效率。

### 1.2 核心价值

- **自然语言交互**：用户无需学习 SQL，用日常语言提问即可获得数据
- **多轮对话**：支持追问和深入分析，逐步细化数据洞察
- **灵活配置**：用户可自行配置数据源，无需依赖 IT 部门
- **数据持久化**：用户、数据源、会话记录全部持久化存储
- **用户认证**：完整的登录注册系统，JWT 令牌认证

### 1.3 目标用户

| 用户群体 | 使用场景 | 核心诉求 |
|---------|---------|---------|
| 业务分析师 | 日常数据查询、报表制作 | 快速获取数据，减少等待时间 |
| 产品经理 | 用户行为分析、指标监控 | 自助查询，不依赖数据团队 |
| 管理层 | 经营数据查看、决策支持 | 即时获取关键指标 |
| 运营人员 | 活动效果分析、用户画像 | 灵活查询，快速迭代 |

### 1.4 版本范围 (v3.0)

| 功能 | 是否支持 |
|-----|---------|
| 自然语言转 SQL | ✅ |
| 多轮对话 | ✅ |

| 数据可视化 | ✅ |
| 数据导出 | ✅ |
| 持久化存储 | ✅ |
| 用户认证 | ✅ |

## 2. 功能需求详述

### 2.1 用户认证

#### 2.1.1 功能描述

提供完整的用户注册、登录、认证功能，基于 JWT 令牌进行身份验证。

#### 2.1.2 功能列表

| 功能 | 描述 |
|------|------|
| 用户注册 | 用户名、密码、邮箱注册 |
| 用户登录 | 用户名 + 密码登录，返回 JWT token |
| 个人信息 | 查看当前用户信息 |
| 修改密码 | 已登录用户修改密码 |
| Token 管理 | JWT token 有效期 24 小时 |

#### 2.1.3 安全机制

- 密码 bcrypt 加密存储
- JWT token 签名验证
- HTTPS 传输加密

### 2.2 数据源配置

#### 2.2.1 功能描述

用户可在前端配置数据库连接信息，系统支持多种数据库类型。

#### 2.2.2 支持的数据库

| 数据库类型 | 优先级 | 备注 |
|-----------|-------|------|
| MySQL | P0 | 首版支持 |
| PostgreSQL | P0 | 首版支持 |
| SQLite | P1 | 首版支持（用于测试） |

#### 2.2.3 配置项

```json
{
  "name": "数据源名称",
  "type": "mysql|postgresql|sqlite",
  "host": "数据库主机地址",
  "port": "端口号",
  "database": "数据库名称",
  "username": "用户名",
  "password": "密码（加密存储）"
}
```

#### 2.2.4 验证规则

- 连接测试：保存前验证连接有效性
- 权限检查：确保用户具有 SELECT 权限
- 安全存储：密码 Fernet 加密存储（可选）

### 2.3 自然语言查询

#### 2.3.1 功能描述

用户输入自然语言问题，系统自动转换为 SQL 并执行，返回查询结果。

#### 2.3.2 查询流程

```
用户输入 → 意图理解 → Schema 检索 → SQL 生成 → 执行查询 → 结果展示
```

#### 2.3.3 查询类型支持

| 查询类型 | 示例 | 支持程度 |
|---------|-----|---------|
| 简单查询 | "查询所有用户" | ✅ |
| 聚合查询 | "统计每个部门的员工数" | ✅ |
| 条件筛选 | "查询销售额大于 1 万的订单" | ✅ |
| 多表关联 | "查询每个用户的订单总额" | ✅ |
| 排序分页 | "销售额前 10 的产品" | ✅ |
| 时间查询 | "上个月的订单量" | ✅ |

#### 2.3.4 安全机制

- **SQL 注入防护**：只允许 SELECT 语句
- **敏感表保护**：可配置禁止查询的表
- **结果数量限制**：默认最大返回 1000 条
- **查询超时**：默认 30 秒超时

### 2.4 多轮对话

#### 2.4.1 功能描述

用户可对查询结果进行追问，系统基于上下文理解用户意图。

#### 2.4.2 对话能力

| 对话类型 | 示例 |
|---------|-----|
| 结果筛选 | "只要北京的" |
| 维度下钻 | "按月份拆分" |
| 指标对比 | "和去年同期对比" |
| 追加条件 | "再加一个条件：状态为已完成" |
| 查询修正 | "我要的是订单数不是金额" |

#### 2.4.3 上下文管理

- **会话隔离**：每个数据源独立会话
- **上下文窗口**：保留最近 10 轮对话
- **持久化存储**：会话和消息存储于 MySQL

### 2.5 数据可视化

#### 2.5.1 功能描述

当用户明确要求可视化时（如"用柱状图展示"），系统自动生成 ECharts 图表。

#### 2.5.2 支持的图表类型

| 图表类型 | 说明 |
|---------|------|
| 柱状图 | 分类数据对比 |
| 折线图 | 趋势数据展示 |
| 饼图 | 占比数据展示 |
| 散点图 | 相关性分析 |
| 面积图 | 累积趋势展示 |

#### 2.5.3 触发条件

- 用户明确要求（如"用柱状图展示"）
- 不自动触发，保持智能体自由度

### 2.6 数据导出

#### 2.6.1 功能描述

当用户要求导出时，系统生成 CSV 或 Excel 文件供下载。

#### 2.6.2 支持格式

| 格式 | 说明 |
|------|------|
| CSV | 逗号分隔值文件 |
| Excel (.xlsx) | Excel 工作簿 |

#### 2.6.3 导出流程

```
用户要求 → Agent 调用 export_data → 生成文件 → 返回下载链接 → 用户下载
```

## 5. 数据库设计

### 5.1 数据库表结构

#### 5.1.1 用户表 (users)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键，UUID |
| username | VARCHAR(50) | 用户名，唯一 |
| password_hash | VARCHAR(255) | bcrypt 密码哈希 |
| email | VARCHAR(100) | 邮箱 |
| status | TINYINT | 状态：1-正常，0-禁用 |
| created_at | DATETIME | 创建时间 |

#### 5.1.2 数据源表 (datasources)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键，UUID |
| user_id | VARCHAR(36) | 外键，关联 users |
| name | VARCHAR(100) | 数据源名称 |
| type | VARCHAR(20) | 类型：mysql/postgresql/sqlite |
| host | VARCHAR(255) | 主机地址 |
| port | INT | 端口 |
| database_name | VARCHAR(100) | 数据库名 |
| db_username | VARCHAR(100) | 数据库用户名 |
| db_password | VARCHAR(255) | 数据库密码 |
| created_at | DATETIME | 创建时间 |

#### 5.1.3 会话表 (sessions)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键，UUID |
| user_id | VARCHAR(36) | 外键，关联 users |
| datasource_id | VARCHAR(36) | 数据源 ID |
| created_at | DATETIME | 创建时间 |
| last_active_at | DATETIME | 最后活跃时间 |

#### 5.1.4 消息表 (messages)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) | 主键，UUID |
| session_id | VARCHAR(36) | 外键，关联 sessions |
| role | VARCHAR(20) | 角色：user/assistant |
| content | TEXT | 消息内容 |
| sql | TEXT | 生成的 SQL |
| result | LONGTEXT | 查询结果 (JSON) |
| created_at | DATETIME | 创建时间 |

### 5.2 ER 图

```
┌─────────────────┐       ┌─────────────────┐
│     users       │       │   datasources   │
├─────────────────┤       ├─────────────────┤
│ id (PK)         │◄──────│ id (PK)         │
│ username        │       │ user_id (FK)    │
│ password_hash   │       │ name            │
│ email           │       │ type            │
│ status          │       │ host            │
│ created_at      │       │ port            │
└─────────────────┘       │ database_name   │
                          │ db_username     │
                          │ db_password     │
                          └─────────────────┘
        ▲
        │
        │
┌─────────────────┐       ┌─────────────────┐
│    sessions     │       │     messages    │
├─────────────────┤       ├─────────────────┤
│ id (PK)         │◄──────│ id (PK)         │
│ user_id (FK)    │       │ session_id (FK) │
│ datasource_id   │       │ role            │
│ created_at      │       │ content         │
│ last_active_at  │       │ sql             │
└─────────────────┘       │ result          │
                          │ created_at      │
                          └─────────────────┘
```

### 5.3 关系说明

| 关系 | 类型 | 说明 |
|------|------|------|
| users → datasources | 1:N | 一个用户可配置多个数据源 |
| users → sessions | 1:N | 一个用户可创建多个会话 |
| sessions → messages | 1:N | 一个会话包含多条消息 |
| datasources → sessions | N:M | 会话关联数据源 |

### 5.4 级联规则

- 删除用户 → 级联删除该用户的所有 datasources、sessions、messages
- 删除数据源 → 级联删除关联的 sessions 和 messages
- 删除会话 → 级联删除关联的 messages

## 7. 非功能性需求

### 7.1 性能要求

| 指标 | 目标值 | 说明 |
|-----|-------|------|
| API 响应时间 | < 200ms | 不含 AI 处理时间 |
| SQL 生成时间 | < 5s | 简单查询 |
| SQL 执行时间 | < 30s | 含查询超时控制 |
| 并发会话数 | 100 | 单实例支持 |
| JWT Token 有效期 | 24 小时 | 访问令牌有效期 |

### 7.2 安全要求

| 要求 | 说明 |
|-----|------|
| SQL 注入防护 | 只允许 SELECT 语句，禁止其他 DML/DDL |
| 敏感数据脱敏 | 密码、手机号等敏感数据自动脱敏 |
| 连接加密 | 数据库连接使用 SSL/TLS |
| 密码加密 | bcrypt 加密存储 |
| JWT 认证 | 所有 API 需携带有效 token |

### 7.3 可用性要求

| 要求 | 说明 |
|-----|------|
| 错误提示 | 友好的错误信息，指导用户修正 |
| 查询建议 | 主动提供查询优化建议 |
| Schema 提示 | 输入时自动提示表名、字段名 |
| 流式响应 | SSE 实时返回处理状态 |
