## 1. 产品概述

### 1.1 产品定位

SmartNum 是一款智能问数系统，旨在让非技术用户也能通过自然语言查询数据库，降低数据分析门槛，提升数据驱动决策效率。

### 1.2 核心价值

| 价值点 | 说明 |
|--------|------|
| 零门槛 | 无需学习 SQL，用日常语言提问即可获取数据 |
| 智能化 | 基于 LLM 的智能 SQL 生成，支持复杂查询 |
| 高效率 | 流式响应，实时反馈，缩短等待时间 |
| 可视化 | 一键生成图表，直观展示数据洞察 |
| 安全性 | 完善的权限控制和数据保护机制 |

### 1.3 目标用户

| 用户群体 | 使用场景 | 核心诉求 |
|----------|----------|----------|
| 业务分析师 | 日常数据查询、报表制作 | 快速获取数据，减少等待 |
| 产品经理 | 用户行为分析、指标监控 | 自助查询，不依赖数据团队 |
| 管理层 | 经营数据查看、决策支持 | 即时获取关键指标 |
| 运营人员 | 活动效果分析、用户画像 | 灵活查询，快速迭代 |

## 2. 功能模块

### 2.1 用户认证

#### 功能说明

提供完整的用户注册、登录、认证功能，基于 JWT 令牌进行身份验证。

#### 功能列表

| 功能 | 描述 |
|------|------|
| 用户注册 | 用户名、密码、邮箱注册 |
| 用户登录 | 用户名 + 密码登录，返回 JWT Token |
| Token 管理 | Token 有效期 24 小时，自动续期 |
| 修改密码 | 已登录用户修改密码 |

#### 安全机制

- 密码 bcrypt 加密存储
- JWT Token 签名验证
- HTTPS 传输加密

### 2.2 数据源管理

#### 功能说明

用户可配置和管理数据库连接，支持多种数据库类型。

#### 支持的数据库

| 数据库 | 驱动 | 说明 |
|--------|------|------|
| MySQL | aiomysql | 主要支持 |
| PostgreSQL | asyncpg | 完全支持 |
| SQLite | aiosqlite | 测试支持 |

#### 配置项

| 字段 | 说明 | 必填 |
|------|------|------|
| 名称 | 数据源显示名称 | 是 |
| 类型 | mysql / postgresql / sqlite | 是 |
| 主机地址 | 数据库服务器地址 | 是 |
| 端口 | 数据库端口 | 是 |
| 数据库名 | 数据库名称 | 是 |
| 用户名 | 数据库用户名 | 是 |
| 密码 | 数据库密码 | 是 |
| Schema | PostgreSQL Schema 名 | 否 |

#### 功能列表

- 添加数据源
- 测试连接
- 删除数据源
- 查看数据库 Schema

### 2.3 知识库管理

#### 功能说明

用户可上传业务知识文件，帮助 AI 理解业务规则、数据字典和注意事项，提升查询准确性。

#### 支持的文件格式

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| 纯文本 | .txt | 最简单的格式 |
| Markdown | .md | 支持格式化内容 |
| Word | .docx | 自动提取文本 |
| PDF | .pdf | 自动提取文本 |

#### 知识库隔离

知识库按数据源隔离，每个数据源有独立的知识库：

```
data/knowledge/
├── {datasource_id_1}/     # 数据源 A 的知识库
│   ├── raw/               # 原始文件
│   └── processed/         # 处理后的文本
├── {datasource_id_2}/     # 数据源 B 的知识库
│   └── ...
```

#### 功能列表

| 功能 | 描述 |
|------|------|
| 上传文件 | 拖拽或选择文件上传 |
| 文件列表 | 查看已上传的文件 |
| 文件预览 | 预览文件内容 |
| 删除文件 | 删除不需要的文件 |

#### AI 知识探索

AI 通过 `explore_knowledge` 工具自由探索知识库：

| 命令 | 用途 |
|------|------|
| `ls -la` | 列出所有知识文件 |
| `cat *.txt` | 阅读所有文件内容 |
| `grep "关键词" .` | 搜索特定内容 |

#### 使用场景

| 场景 | 知识文件示例 |
|------|-------------|
| 业务规则 | "查询用户时，不要排除已删除的用户" |
| 数据字典 | "status 字段：0-禁用，1-启用，2-待审核" |
| 注意事项 | "order 表数据量大，查询时必须加时间范围条件" |
| 常用查询 | "月报查询的 SQL 模板..." |

### 2.4 智能对话

#### 功能说明

用户输入自然语言问题，系统自动转换为 SQL 并执行，返回查询结果。

#### 查询流程

```
用户输入 → 意图理解 → Schema 检索 → SQL 生成 → 执行查询 → 结果展示
```

#### 查询类型支持

| 查询类型 | 示例 | 支持程度 |
|----------|------|----------|
| 简单查询 | "查询所有用户" | ✅ |
| 聚合查询 | "统计每个部门的员工数" | ✅ |
| 条件筛选 | "查询销售额大于 1 万的订单" | ✅ |
| 多表关联 | "查询每个用户的订单总额" | ✅ |
| 排序分页 | "销售额前 10 的产品" | ✅ |
| 时间查询 | "上个月的订单量" | ✅ |

#### 多轮对话

系统支持基于上下文的追问：

| 对话类型 | 示例 |
|----------|------|
| 结果筛选 | "只要北京的" |
| 维度下钻 | "按月份拆分" |
| 指标对比 | "和去年同期对比" |
| 追加条件 | "再加一个条件：状态为已完成" |
| 查询修正 | "我要的是订单数不是金额" |

#### 智能体执行步骤

系统实时展示智能体执行步骤，包括：

| 步骤类型 | 说明 |
|----------|------|
| tool_call | 工具调用（列出数据表、获取表结构、执行 SQL 等） |
| tool_result | 工具返回结果 |

### 2.5 数据可视化

#### 功能说明

当用户要求可视化时，系统自动生成 ECharts 图表。

#### 支持的图表类型

| 图表类型 | 适用场景 | 触发示例 |
|----------|----------|----------|
| 柱状图 | 分类数据对比 | "用柱状图展示各部门人数" |
| 折线图 | 趋势数据展示 | "画一条折线图展示月度销售额" |
| 饼图 | 占比数据展示 | "用饼图展示各渠道占比" |
| 散点图 | 相关性分析 | "画散点图分析价格和销量的关系" |
| 面积图 | 累积趋势展示 | "用面积图展示累计用户增长" |

#### 触发条件

- 用户明确要求（如"用柱状图展示"）
- 不自动触发，保持智能体自由度

### 2.6 数据导出

#### 功能说明

当用户要求导出时，系统生成 CSV 或 Excel 文件供下载。

#### 支持格式

| 格式 | 说明 | 触发示例 |
|------|------|----------|
| CSV | 逗号分隔值文件 | "帮我导出 CSV 格式" |
| Excel | Excel 工作簿 | "导出 Excel 表格" |

#### 导出流程

```
用户要求 → Agent 调用 export_data 工具 → 生成文件 → 显示下载卡片 → 用户点击下载
```

### 2.7 仪表盘

#### 功能说明

系统首页仪表盘展示关键指标，方便用户快速了解数据概况。

#### 展示指标

| 指标 | 说明 |
|------|------|
| 数据源数量 | 当前用户配置的数据源总数 |
| 会话数 | 当前用户的会话总数 |
| 查询次数 | 所有会话的消息总数 |

#### 快捷入口

- 添加数据源
- 开始对话
- 最近会话列表

## 3. 使用指南

### 3.1 用户注册/登录

1. 访问前端界面，点击「注册」创建账号
2. 填写用户名、密码、邮箱
3. 注册成功后自动登录
4. 已有账号可直接登录

### 3.2 配置数据源

1. 登录后，点击「添加数据源」或导航至数据源页面
2. 选择数据库类型（MySQL/PostgreSQL/SQLite）
3. 填写连接信息
4. 点击「测试连接」验证配置
5. 测试成功后点击「保存」

### 3.3 开始对话

1. 在数据源列表中选择一个数据源
2. 点击「开始对话」进入聊天界面
3. 输入自然语言问题
4. 查看智能体执行步骤和查询结果

### 3.4 多轮追问示例

```
用户：查询上个月的销售数据
系统：[返回销售数据表格]

用户：按地区拆分
系统：[返回按地区分组的销售数据]

用户：只要北京的
系统：[返回北京地区的销售数据]

用户：用柱状图展示
系统：[显示柱状图]
```

### 3.5 数据导出示例

```
用户：帮我导出这份表格
系统：[显示下载卡片，点击下载]
```

## 4. API 接口

### 4.1 接口概览

| 模块 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 认证 | POST | /api/auth/register | 用户注册 |
| 认证 | POST | /api/auth/login | 用户登录 |
| 认证 | GET | /api/auth/me | 获取当前用户 |
| 认证 | POST | /api/auth/change-password | 修改密码 |
| 数据源 | GET | /api/datasources | 获取数据源列表 |
| 数据源 | POST | /api/datasources | 添加数据源 |
| 数据源 | POST | /api/datasources/test | 测试连接 |
| 数据源 | DELETE | /api/datasources/{id} | 删除数据源 |
| 数据源 | GET | /api/datasources/{id}/schema | 获取 Schema |
| 会话 | GET | /api/sessions | 获取会话列表 |
| 会话 | POST | /api/sessions | 创建会话 |
| 会话 | DELETE | /api/sessions/{id} | 删除会话 |
| 会话 | GET | /api/sessions/{id}/messages | 获取对话历史 |
| 会话 | POST | /api/sessions/{id}/messages/stream | 发送消息 (SSE) |
| 会话 | GET | /api/sessions/export/{downloadId} | 下载导出文件 |

### 4.2 认证接口示例

#### 用户登录

**请求**
```http
POST /api/auth/login
Content-Type: application/json

{
  "username": "admin",
  "password": "password123"
}
```

**响应**
```json
{
  "user_id": "usr_abc123",
  "username": "admin",
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
```

### 4.3 流式消息接口

**请求**
```http
POST /api/sessions/{session_id}/messages/stream
Content-Type: application/json
Authorization: Bearer {token}

{
  "content": "查询销售额前 10 的产品"
}
```

**响应 (SSE)**
```
event: tool_call
data: {"type": "tool_call", "tool": "list_tables", "name": "list_tables"}

event: tool_result
data: {"type": "tool_result", "tool": "list_tables", "output": "..."}

event: message
data: {"type": "message", "content": "根据查询结果..."}

event: done
data: {"type": "done"}
```

## 6. 部署指南

### 6.1 环境变量

创建 `.env` 文件并配置：

```env
# LLM 配置 (必需)
LLM_MODEL_NAME=gpt-4
LLM_API_KEY=your-api-key
LLM_BASE_URL=https://api.openai.com/v1

# JWT 配置
SECRET_KEY=your-secret-key
ACCESS_TOKEN_EXPIRE_MINUTES=1440

# MySQL 配置 (必需)
DB_HOST=localhost
DB_PORT=3306
DB_NAME=smartnum
DB_USERNAME=root
DB_PASSWORD=your-password

# 查询配置 (可选)
MAX_RESULT_ROWS=1000
QUERY_TIMEOUT=30
```

### 6.2 数据库初始化

```bash
mysql -u root -p -e "CREATE DATABASE smartnum CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
mysql -u root -p smartnum < init_db.sql
```

### 6.3 启动服务

```bash
# 后端
uvicorn app.main:app --host 0.0.0.0 --port 8000

# 前端构建
cd frontend
npm run build
```

## 7. 常见问题

### Q: 如何添加新的数据源？

A: 登录后进入「数据源」页面，点击右上角「添加数据源」按钮，填写数据库连接信息后测试并保存。

### Q: 查询结果为空怎么办？

A: 可能原因：
1. 查询条件过于严格
2. 数据库中确实没有符合条件的数据
3. 表名或字段名输入错误

建议检查查询条件，或让系统列出可用的表和字段。

### Q: 如何查看生成的 SQL？

A: 在对话界面中，展开「智能体执行步骤」面板，点击「run_sql」步骤即可查看生成的 SQL 语句。

### Q: 支持哪些数据库？

A: 目前支持 MySQL、PostgreSQL、SQLite 三种数据库。

### Q: 数据是否安全？

A: 系统采用多层安全机制：
1. 密码 bcrypt 加密存储
2. JWT Token 认证
3. SQL 注入防护（只允许 SELECT 语句）
4. 查询超时保护
