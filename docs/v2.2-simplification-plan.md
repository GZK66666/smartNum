# SmartNum v2.2 简化重构计划

## 背景

当前 v2.1 架构中，智能体需要决定何时调用 `present_table` 和 `present_chart` 工具来展示数据，这导致：
1. 智能体行为不稳定，有时不调用展示工具
2. prompt 中关于展示工具的描述很复杂
3. 前端需要处理多种 content block 类型
4. 用户只需要智能体回答问题，不需要复杂的展示控制

## 优化目标

### 1. 简化智能体输出
- **去除** `present_table` 和 `present_chart` 工具
- **简化** system prompt，去除关于展示工具的描述
- 智能体只需要用自然语言回答用户问题
- 数据结果直接以 Markdown 表格形式输出

### 2. 细化 Schema 查询工具
实现上下文渐进式加载：

| 新工具 | 功能 | 参数 |
|--------|------|------|
| `list_tables` | 列出所有表名和注释 | `datasource_id` |
| `get_table_schema` | 获取单张表的详细结构 | `datasource_id`, `table_name` |
| `run_sql` | 执行 SQL 查询 | `datasource_id`, `sql`, `limit` |

**优势**：
- 首次查询只加载表名列表，token 消耗小
- 智能体按需查询具体表的 schema
- 避免一次性加载所有表结构导致上下文溢出

## 实施步骤

### Step 1: 细化 Schema 查询工具

**文件**: `app/services/agent_service.py`

#### 1.1 新增 `list_tables` 工具

```python
def list_tables(datasource_id: str) -> str:
    """
    列出数据库中所有表的名称和注释。

    在查询数据前，先调用此工具了解有哪些表可用。
    返回简洁的表列表，不包含列信息。

    Args:
        datasource_id: 数据源ID

    Returns:
        表名和注释列表（Markdown 格式）
    """
```

#### 1.2 新增 `get_table_schema` 工具

```python
def get_table_schema(
    datasource_id: str,
    table_name: str,
) -> str:
    """
    获取指定表的详细结构信息。

    当你需要了解某个表的列名、类型、注释等信息时调用此工具。
    建议只查询与用户问题相关的表，避免加载过多信息。

    Args:
        datasource_id: 数据源ID
        table_name: 表名

    Returns:
        表结构信息（Markdown 格式），包含列名、类型、注释等
    """
```

#### 1.3 保留 `run_sql` 工具

保持不变，用于执行 SQL 查询。

#### 1.4 删除 `present_table` 和 `present_chart` 工具

### Step 2: 简化 System Prompt

**文件**: `app/services/agent_service.py`

**新 Prompt**:

```python
SYSTEM_PROMPT = """你是 SmartNum 数据分析助手，专门帮助用户查询和分析数据库中的数据。

## 工作流程

1. **理解需求**：分析用户的问题，确定需要查询什么数据
2. **探索结构**：
   - 先用 `list_tables` 了解有哪些表可用
   - 再用 `get_table_schema` 查询相关表的结构
3. **执行查询**：使用 `run_sql` 执行 SQL 查询获取数据
4. **回答问题**：用自然语言回答用户问题，数据用 Markdown 表格展示

## 工具说明

### list_tables
列出数据库中所有的表名和注释。在开始查询前先了解有哪些表可用。

### get_table_schema
获取指定表的详细结构（列名、类型、注释）。只查询与用户问题相关的表。

### run_sql
执行 SQL SELECT 查询并返回结果。只支持 SELECT 语句。

## 输出规则

1. 数据结果用 Markdown 表格展示
2. 用简洁的自然语言解读关键发现
3. 回答用户的具体问题

## 安全规则

- 只生成 SELECT 语句，禁止 DELETE/UPDATE/INSERT
- 不查询敏感数据（如密码、身份证号）
"""
```

### Step 3: 简化 SSE 事件流

**文件**: `app/services/agent_service.py`

#### 3.1 删除的事件类型
- `ContentBlockEvent` (content_block)
- 与 `present_table`/`present_chart` 相关的处理逻辑

#### 3.2 保留的事件类型
- `ThinkingEvent` - 思考过程
- `ToolCallEvent` - 工具调用开始
- `ToolResultEvent` - 工具调用结果
- `SQLGenerationEvent` - SQL 生成
- `SQLExecutionEvent` - SQL 执行状态
- `MessageEvent` - 文本消息（智能体回复）
- `ErrorEvent` - 错误信息
- `DoneEvent` - 处理完成

#### 3.3 简化最终结果结构

```python
final_result = {
    "content": "",      # 智能体的文本回复
    "sql": None,        # 执行的 SQL（如果有）
    "error": None,      # 错误信息（如果有）
}
```

### Step 4: 更新前端展示

**文件**: `frontend/src/pages/ChatPage.tsx`

#### 4.1 简化 ContentBlockRenderer

移除 `table` 和 `chart` 类型的处理，只保留 `text` 类型。

```typescript
function ContentBlockRenderer({ block }: { block: ContentBlock }) {
  if (block.type === 'text') {
    return (
      <div className="prose prose-invert prose-sm max-w-none text-slate-200 mb-3">
        <ReactMarkdown remarkPlugins={[remarkGfm]}>
          {block.content}
        </ReactMarkdown>
      </div>
    );
  }
  return null;
}
```

#### 4.2 简化消息处理

- 移除 `blocks` 数组的复杂处理
- 智能体回复直接作为文本消息展示
- Markdown 表格自动渲染
